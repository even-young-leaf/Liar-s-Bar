# AI逻辑全面优化文档

## 优化概述

将原本简单的规则AI升级为**多策略智能AI系统**，实现了4种不同性格的AI策略，并增加了完善的决策逻辑和错误处理。

---

## 原有问题

### 1. 出牌逻辑问题
- ❌ 无脑打手牌前N张，不看牌面
- ❌ 永远声称打目标牌，即使手里完全没有
- ❌ 不考虑WILD牌的使用
- ❌ 不考虑手牌剩余数量策略

### 2. 质疑逻辑问题
- ❌ 质疑条件过于苛刻（对方出2张+自己有2张才质疑）
- ❌ 几乎从不质疑
- ❌ 不考虑对方手牌数量
- ❌ 不考虑对方历史说谎率
- ❌ 不考虑Bristle角色优势（可质疑2次）

### 3. 决策流程问题
- ❌ 永远不会主动跳过
- ❌ 没有战略性思考
- ❌ 行为模式单一可预测

## 新实现的功能

### 1. 四种AI策略类型 🎭

#### 保守型 (Conservative)
**性格特点：**
- 少说谎，优先打真牌
- 更倾向于质疑别人
- 手里没真牌时会跳过
- 出牌数量少（1-2张）

**适用场景：**
- 适合防守型玩法
- 降低被质疑风险
- 稳扎稳打

**质疑概率：** 基础概率 × 1.3

---

#### 激进型 (Aggressive)
**性格特点：**
- 经常说谎，混合打牌
- 较少质疑别人（专注自己出牌）
- 几乎不跳过
- 出牌数量多（2-3张）

**适用场景：**
- 快速出牌，压迫对手
- 制造不确定性
- 高风险高收益

**质疑概率：** 基础概率 × 0.7

---

#### 平衡型 (Balanced)
**性格特点：**
- 优先打真牌，必要时说谎
- 适度质疑
- 偶尔跳过
- 出牌数量灵活（1-3张）

**适用场景：**
- 最接近人类玩家
- 综合考虑风险和收益
- 难以预测

**质疑概率：** 基础概率 × 1.0

---

#### 随机型 (Random)
**性格特点：**
- 完全不可预测
- 随机选牌、随机数量
- 随机质疑和跳过
- 增加游戏趣味性

**适用场景：**
- 制造混乱
- 防止玩家摸清AI套路
- 娱乐性强

**质疑概率：** 完全随机

### 2. 智能决策系统 🧠

#### 手牌分析
```go
func (ai *aiStrategy) analyzeHand() (targetCards, wildCards, otherCards []int)
```
- 自动分类手牌：目标牌、WILD牌、其他牌
- 为后续决策提供数据基础

#### 说谎风险计算
```go
func (ai *aiStrategy) calculateLyingRisk(targetCards, wildCards []int) float64
```
- 考虑手里真牌数量
- 考虑场上存活玩家数量
- 玩家越少，被质疑风险越高

#### 跳过决策
```go
func (ai *aiStrategy) shouldSkipTurn() bool
```
根据策略类型和手牌情况决定：
- **保守型：** 没真牌时30%跳过
- **激进型：** 几乎不跳过
- **平衡型：** 没真牌且手牌少时15%跳过
- **随机型：** 10%随机跳过

#### 出牌数量决策
```go
func (ai *aiStrategy) decidePlayCount() int
```
根据策略和手牌数量动态调整：
- **保守型：** 1-2张（降低风险）
- **激进型：** 2-3张（快速出牌）
- **平衡型：** 1-3张（灵活调整）
- **随机型：** 完全随机

#### 选牌策略
```go
func (ai *aiStrategy) selectCards(playCount int) []int
```
不同策略的选牌逻辑：
- **保守型：** 目标牌 → WILD → 其他牌（手牌多时才说谎）
- **激进型：** 50%概率混入假牌
- **平衡型：** 目标牌 → WILD → 其他牌
- **随机型：** 完全随机打牌

### 3. 高级质疑系统 🎯

#### 多因素质疑概率计算

**因素1：牌面数量对比**
```
对方出3张 + 我有4张 → 80%
对方出3张 + 我有3张 → 60%
对方出3张 + 我有2张 → 30%
对方出2张 + 我有4张 → 50%
对方出2张 + 我有3张 → 30%
对方出1张 + 我有5张 → 20%
```

**因素2：对方手牌数量**
- 对方手牌 ≤ 2张 且出 ≥ 2张 → +15%
- 手牌少可能着急出牌，容易说谎

**因素3：历史说谎率**
- 对方说谎率 > 50% → +10%
- 跟踪每个玩家的历史行为

**因素4：角色技能**
- Bristle角色：每轮可质疑2次
- 自动检查质疑次数限制

**策略调整：**
- 保守型：× 1.3
- 激进型：× 0.7
- 平衡型：× 1.0
- 随机型：完全随机

**概率限制：** 最终概率限制在 [0.0, 0.9] 范围内

### 4. 完善的错误处理 🛡️

#### 索引有效性验证
```go
for _, idx := range indices {
    if idx < 0 || idx >= len(player.Hand) {
        log.Printf("AI selected invalid card index...")
        r.executePass(player.ID)
        return
    }
}
```

#### 出牌失败保护
```go
err := r.State.PlayCard(player.ID, indices, r.State.TargetCard)
if err != nil {
    log.Printf("AI play card error for player %d: %v", player.ID, err)
    r.executePass(player.ID)
    return
}
```

#### 空选牌保护
```go
if len(indices) == 0 {
    r.executePass(player.ID)
    return
}
```

## 技术亮点

### 1. 策略随机化
每次创建AI决策时随机选择策略类型，确保游戏多样性：
```go
strategyType := AIStrategyType(rand.Intn(4))
```

### 2. 概率驱动决策
所有决策都基于概率模型，避免固定套路：
```go
return rand.Float64() < baseProbability
```

### 3. 多维度风险评估
综合考虑手牌、对手状态、历史数据：
- 手牌组成
- 场上玩家数量
- 对方手牌数量
- 历史说谎率

### 4. 自适应行为
根据游戏状态动态调整：
- 手牌少 → 更保守
- 对方说谎多 → 更倾向质疑
- 存活玩家少 → 降低说谎频率
